Submit a file type proposal

Please fill in the fields below and press the Submit button to add your file type to the database as a proposal. See the description of the process to find out what will happen after you made your submission.

Notes:

  • The ID may only contain letters, numbers, underscore and hyphen. You should use capital letters. File type IDs which only differ by their case are considered duplicates.
  • The extension should be in lowercase and has the same restrictions as the ID. It may not contain a dot.
    When a file type may have several extensions (like AIFF -> aif and aiff), provide the most common extension and write the alternatives in the comment field. Currently, Java Sound only supports one extension per file type.
    It is only of descriptive nature.
  • The short description provides a long (technical) name for the file type. Do not include specification details.
  • Contact is the person or company who submits the file type to the database. It is not necessarily the creator of the file format.
  • The comment field should contain additional specifications how to use the file type, like e.g. special considerations of supported AudioFormats. Do not include technical description how to implement AudioFileWriters or AudioFileReaders. You cannot include html code, but complete http and ftp URL's and email adresses are linked automatically.
  • After pressing the Submit button, you'll have the possibility to verify your proposal. Further changes can be requested by sending an email to Florian or Matthias.
  • With pressing the Submit button you agree that your provided information is made publicly available.
